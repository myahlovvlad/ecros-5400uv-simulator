# SPEC-MULTI-001 — Многоволновой анализ и единый operational workflow

## 1. Цель

Добавить в симулятор ЭКРОС-5400УФ режим `МНОГОВОЛН. АНАЛИЗ` и привести режимы измерений к единому operational workflow.

Базовый workflow для режимов измерений:

1. Выбор режима измерения.
2. Установка параметров выполнения измерений.
3. Обнуление.
4. Измерение.
5. Табличное представление результатов.
6. Визуализация результатов только для режимов `КОЛИЧ. АНАЛИЗ` и `КИНЕТИКА`.
7. Сохранение, экспорт, печать данных.

## 2. Новая целевая структура режимов

### 2.1 Главное меню

Главное меню должно содержать:

1. `ФОТОМЕТРИЯ`
2. `КОЛИЧ. АНАЛИЗ`
3. `КИНЕТИКА`
4. `МНОГОВОЛН. АНАЛИЗ`
5. `НАСТРОЙКИ`

### 2.2 Параметры режимов

#### Фотометрия

- длина волны: 190–1100 нм
- измеряемая величина: `A`, `%T`, `E`
- число параллельных измерений: 1–9

#### Колич. анализ / Градуировочный график

- длина волны: 190–1100 нм
- измеряемая величина: `A`, `%T`, `E`
- число параллельных измерений: 1–9
- число стандартов: 1–9

#### Колич. анализ / Метод коэффициентов

- длина волны: 190–1100 нм
- измеряемая величина: `A`, `%T`, `E`
- число параллельных измерений: 1–9
- коэффициенты `K` и `B`

#### Кинетика

- длина волны: 190–1100 нм
- измеряемая величина: `A`, `%T`, `E`
- время измерения: 0–600 с

#### Многоволновой анализ

- число длин волн: 2–4
- последовательный ввод длин волн `λ1..λ4`
- диапазон каждой длины волны: 190–1100 нм
- измеряемая величина: `A`, `%T`, `E`

## 6. Definition of Done

Изменение считается завершённым, если выполнены все условия:

1. В главном меню появился пункт `МНОГОВОЛН. АНАЛИЗ`.
2. Пользователь может задать число длин волн 2–4.
3. Пользователь может последовательно ввести `λ1..λN`.
4. После `ZERO` и `START` выполняется серия измерений по всем выбранным длинам волн.
5. Результаты выводятся в виде таблицы на LCD.
6. Результаты можно сохранить в файловую группу `МНОГОВОЛНОВЫЙ`.
7. USB preview поддерживает экспорт многоволнового файла.
8. Навигация по `ESC`, `ENTER`, `UP`, `DOWN`, `FILE`, `ZERO`, `START/STOP` не ломает существующие режимы.
9. `npm run build` и `npm test -- --run` проходят успешно.

## 8. Ограничения текущего проекта

Существующая архитектура уже позволяет добавить режим, но ещё не реализует полностью единый workflow на уровне общей state machine. Поэтому внедрение должно идти в два этапа:

### Этап A

- добавить `МНОГОВОЛН. АНАЛИЗ`
- добавить новую файловую группу
- синхронизировать экраны и рендеринг

### Этап B

- унифицировать параметры всех режимов через общий measurement workflow
- убрать разрастание `switch(device.screen)` и перевести его на map handlers
- унифицировать файловое меню и печать
